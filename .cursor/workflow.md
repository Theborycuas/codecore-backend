# Workflow

Para cada paso:

1. Analizar documentación existente.
2. Auditar alineamiento código ↔ documentación.
3. Implementar cambios mínimos.
4. Ejecutar pruebas.
5. Actualizar documentación.
6. Generar auditoría en docs/audits.
7. Entregar resumen corto.

No saltar pasos.

No marcar una tarea como terminada sin ejecutar pruebas.
